PP Sheet — Pack Parameters

Pack Rounding Logic

The PP sheet defines boards per pack for each WW product. Pack size in linear meters is calculated as:

Pack Size = (board_mm / 10) × boards_per_pack(WW) Example: WW = 1934, boards_per_pack = 120 board_mm = 42 Pack Size = (42/10) × 120 = 504 LM

3. The Replenishment Formula

Classic safety-stock reorder point with pack rounding • Distinct logic for MTS vs MTO items

Make to Stock (MTS)
Step 1: Compute Thresholds Safety Stock = dimension_mm × demand_factor Target Stock = demand_factor × (dimension_mm + 0.75) Step 2: Check Availability On Hand = SUMIFS(AX inventory snapshot) Available = On Hand − Open SO (current month) Step 3: Trigger Decision IF Available ≤ Safety Stock: Produce = ROUNDUP( (Target − Available) / Pack Size ) × Pack Size ELSE: Produce = 0
Pack rounding is critical. Production quantities must be exact multiples of pack size to match physical stacking and dryer load constraints.
Make to Order (MTO)

Zero Safety Stock

MTO items have Safety Stock = 0 and Target Stock = 0. Production only triggers against confirmed sales demand. No speculative inventory build.

Rolling Forward (M+1 through M+5)
For each forward month: Available[M+n] = Available[M+n−1] − Demand[M+n] + Production[M+n] IF Available[M+n] ≤ Safety Stock: Trigger replenishment (same formula) Slow Mover Flag: IF stock > ½ annual_demand → "SLOW" ELSE → "OK" MOH = inventory / (annual_demand / 12)
Existing production orders are subtracted before triggering new production. The formula avoids double-ordering.

PlndProd Formula (Two-Month Lookahead)
PlndProd[month] = IF ( Inv[month−1] − OpenSales[month] − OpenSales[month+1] ← lookahead − FC[month] − US_FC[month] + Prod[month] < Safety_Stock × Repl_Factor ) THEN: MAX(Target − projected_avail, 0) ELSE: 0

Inv[month] = Inv[month−1] − OpenSales[month] − FC[month] − US_FC[month] + Prod[month] (confirmed) + PlndProd[month] (proposed)

5. What Works Well — Preserve in Odoo

The RAP MPS workbook embeds proven operational discipline • These patterns must survive the migration

  • Safety stock reorder point — Classic trigger: when projected inventory drops below safety stock, produce up to target level
  • Pack rounding to production multiples — Quantities always rounded up to full packs, matching physical stacking constraints
  • MTS vs MTO distinction — Per-SKU planning method: MTO items get zero safety stock, only produce against demand
  • Slow mover detection (SLOW flag) — Flags SKUs where stock exceeds half of annual demand — FIFO discipline mechanism
  • US forecast overlay with seasonality — Separate quarterly-split forecast for the US market, additive to base demand
  • Multi-level BOM explosion with waste — FG → RS (×1.10) → RM (×1.05) cascade ensures raw material covers yield loss
  • Rolling 6-month projection — Forward visibility of inventory balance, not just current month reorder
  • Existing PO/MO deduction — Avoids double-ordering by subtracting confirmed production and purchase orders
Key insight: The Excel logic is sound. The problem is not the algorithm — it is the lack of capacity constraints, traceability, and automation. Odoo must replicate this logic AND add what Excel cannot do.

6. What Excel Cannot Do — Odoo Must Add

10 structural limitations of the spreadsheet approach • Each maps to a specific Odoo capability

Limitations
  • No dryer capacity constraint — ignores the primary bottleneck entirely
  • No family batching — plans per SKU independently, not by compatible families
  • No autoclave/dryer load composition — cannot model process pack → load hierarchy
  • No WIP tracking — only sees FG/RS/RM inventory, blind to in-process stock
  • No lot traceability — no link from raw material batch to finished goods
  • No cost integration — quantities only, no FIFO cost flow
  • No multi-entity view — single warehouse, no cross-company planning
  • Static forecast — manual refresh, no live SO pipeline
  • No chemical/mix planning link — cannot trigger mix batches from tank levels
  • Approximate pack rounding — not tied to planning family definitions

Load composition Process Pack → Autoclave Load → Dryer Load
WIP tracking Real-time WIP via stock locations + MO states
Lot traceability Full lot tracking RM → RS → FG
Cost integration FIFO lot cost flows through BOM explosion
Multi-entity Multi-company with entity-specific rules
Live forecast Dynamic from Odoo SO pipeline + CRM
Chemical planning Mix batch MO triggered by tank reorder point

9. Decision Points — CEO/CFO Approval Required

Five open questions that must be resolved before Phase 2 implementation can begin

1  Safety Stock Levels Validation

Current safety stock formulas are embedded in Excel. Business must sign off on the formula (dimension × demand_factor) before it is codified in Odoo reordering rules. Are these levels correct, or do they need recalibration?

2  Growth Rate Assumptions

The Calc engine uses 1.2× while the RAD sheets use 1.1×. Which is the correct growth assumption? This directly impacts safety stock levels and forecast-driven production triggers.

3  US Forecast Ownership

The quarterly split percentages (5/5/5 → 9/9/9 → 10/10/10) are maintained manually. Who owns this forecast in Odoo? US sales team? Supply chain? How often is it updated?

4  Waste Factors Confirmation

RS→FG = 10%, RM→RS = 5%. Are these measured from production data, or are they assumed? Should they vary by product family or remain global? Waste factors drive raw material purchasing.

5  MTO vs MTS Classification Review

Some products currently classified as MTO may benefit from MTS treatment (and vice versa). A full product classification review is needed before Odoo replenishment rules are activated. Reclassification changes safety stock, forecast behaviour, and dryer scheduling priority.

These are not IT decisions. Each one directly impacts inventory investment, service levels, and production capacity utilisation. CEO/CFO sign-off required before Phase 2 development starts.
