Data Visibility — Multi-Company

Which entities a user can see is controlled by their company assignment
How it works
  • Each user has a list of Allowed Companies on their profile
  • Record rules filter data to only show records from those companies
  • Users can switch between their allowed companies via the company switcher
  • This is native Odoo — no custom code needed
Entity Map
Entity Code Location Type
Kebony Inc. INC US Trader
Kebony BNV BNV Belgium Manufacturer
Kebony Holding Holding Belgium Holding
Kebony KAS KAS Norway Manufacturer
Kebony Norge AS NAS Norway Operations
Example Assignments
User: Sarah (US Finance) BU Group: 15 – Finance & Administration Companies: INC only → Sees only Kebony Inc. data → Full accounting rights User: Erik (Norway Production) BU Group: 30 – Production Companies: KAS + NAS → Sees both Norwegian entities → Full manufacturing rights User: Marc (Group CFO) BU Group: 15 – Finance & Administration Companies: ALL (INC + BNV + Holding + KAS + NAS) → Sees all entities → Full accounting rights everywhere
Same BU, different visibility: Sarah and Marc are both BU 15, but Sarah sees only INC while Marc sees everything. The BU defines what they can do. The company list defines where.

How to Assign a User

Step-by-step procedure for IT / HR
New User Setup
  1. Go to Settings → Users & Companies → Users
  2. Create new user (name, email, login)
  3. Scroll to "Kebony Business Units" section
  4. Select the appropriate BU from the dropdown
  5. Set Allowed Companies (which entities they need)
  6. Set Default Company (their primary entity)
  7. Save — all implied permissions are set automatically
That's it. No need to manually set Accounting Manager, Stock User, MRP User, etc. The BU group handles it all via implied groups.
Changing a User's BU
  1. Open user form
  2. Change BU group in the dropdown
  3. Save — old permissions removed, new ones applied
Things to NOT do
  • Don't manually add standard Odoo groups — the BU handles it
  • Don't give Admin rights unless truly needed
  • Don't assign multiple BU groups to one user — pick one
  • Don't use Studio to create new groups — use the code
Admin = God mode. Admin users bypass ALL restrictions. Only give Admin to IT administrators who need system configuration access.

Monthly Security Audit Checklist

Review quarterly (or after any org change)
User Review
# Check Action if failed
1 All active users have a BU group assigned Assign appropriate BU
2 No user has multiple BU groups Remove extra groups
3 Leavers are deactivated (not deleted) Archive user
4 Admin count is minimal (max 2-3) Downgrade to BU group
5 Company assignments match org structure Update allowed companies
Access Review
# Check Action if failed
6 Sensitive buttons (EOM, margin) only visible to Finance Check groups= in views
7 Manufacturing data not accessible to Sales Check ir.model.access
8 No orphaned Studio groups overriding BU Remove Studio groups
Where to Check

User List

Settings → Users & Companies → Users
Filter by "Kebony Business Unit" column

Group Members

Settings → Technical → Groups
Search "Kebony" → click group → Users tab

Access Rights

Settings → Technical → Security → Access Rights
Filter by group to see all model permissions

Record Rules

Settings → Technical → Security → Record Rules
Verify company isolation rules are active
