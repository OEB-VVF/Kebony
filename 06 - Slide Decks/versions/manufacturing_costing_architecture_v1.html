1. Manufacturing Hierarchy

The atomic planning, costing, and valuation unit is the Dryer Load. One Dryer Load = One Manufacturing Order.

Physical Hierarchy
Raw Lot (white wood, received at FIFO) ↓ Process Pack (PP) [physical only] ↓ Autoclave Batch (AC) [5-8 packs, 2-8h cycle] ↓ Dryer Load (DL) [2 AC = 1 DL, 40-70h] ↓ = 1 MO Finished Lot [sellable FG] ↓ (optional) RDK Machining [subcontracting MO]
Odoo Models
Model Accounting? Status
kebony.planning.family No — master data Done
kebony.process.pack No — physical only Done
kebony.autoclave.batch No — physical only Done
kebony.dryer.load Yes — = MO Done
mrp.production Yes — WIP + FG Phase 6
kebony.cost.version Yes — budget layer Done
Key rule: Only products from the same planning family can share an autoclave load or dryer load.

The "Push BoM to Products" button on a planning family creates or updates the BOM for every product assigned to that family.

5. The Closed Loop — Work Centre × GL Account

Each WC cost line links a work centre to a GL account (cost nature) with a rate per hour. This is the key to variance decomposition.

Example: Dryer Work Centre
GL Account Cost Nature €/hour
620000 Personnel (operators) 25.00
610000 Energy (electricity) 12.50
630000 Depreciation (dryer asset) 8.00
611000 Maintenance 3.50
Total Dryer Rate 49.00
Why multiple lines per WC? Each GL account is a separate cost nature. When the actual GL balance is compared to absorbed amount (rate × hours), you get per-nature variance: "Energy was over budget by €2k, but personnel was under."
Variance Decomposition
Budget (from cost version): Dryer rate = €49.00/h Dryer hours = 54h (from family KRRS1) Load volume = 49.4 m³ Budget cost = 49 × 54 / 49.4 = €53.56/m³ Actual (from MO completion): Actual hours = 58h (4h over) Actual GL 620 = €1,600 (vs budget €1,350) Actual GL 610 = €700 (vs budget €675) Variances: Efficiency: (58-54)h × €49/h = €196 unfav. Rate (620): €1,600 - (58 × €25) = €150 unfav. Rate (610): €700 - (58 × €12.5) = -€25 fav.

6. Family Cost Card — Standard Cost per m³

Each family line in a cost version automatically computes the full standard cost breakdown. This is the bridge between planning families and financial control.

Computed Fields (per m³ of FG)
Cost Element Formula Example (KRRS 1")
Scrap factor 1 / (1 − int% − bg%) 1 / (1 − 0.016 − 0.023) = 1.0406
Wood cost / m³ scrap_factor × wood_price 1.0406 × €280 = €291.37
Chemical cost / m³ mix_kg_per_m3 × mix_price_kg 676 × €0.045 = €30.42
Stacking / m³ stack_h × Σ(stack rates) / vol 4.5 × €35 / 49.4 = €3.19
Autoclave / m³ ac_h × Σ(ac rates) / vol 0 × €42 / 49.4 = €0.00
Dryer / m³ dry_h × Σ(dry rates) / vol 54 × €49 / 49.4 = €53.56
Conversion / m³ stack + ac + dryer €3.19 + €0.00 + €53.56 = €56.75
Total std cost / m³ wood + chem + conv €378.54
Data Sources

Hours & Scrap

From family line snapshot (editable override of family master)

WC Rates (€/hour)

Sum of all WC cost lines for each work centre (multi-line per GL account)

Volume (m³)

typical_dryer_load_volume from planning family master

RM Prices

From Product Price lines in the same cost version (wood & ready-mix)

All computed, all stored. Change a WC rate or RM price → family cost card updates automatically. Lock the version → snapshot frozen.

Gap Analysis Logic
Dimension Budget (Cost Version) Actual (FIFO MO)
Wood price Product Price line FIFO purchase price
Wood qty / m³ Scrap factor (family) Actual MO consumption / output m³
Chemical price Product Price line FIFO ready-mix cost
Chemical qty / m³ Mix consumption (family) Actual MO consumption / output m³
WC rate / hour WC Cost Lines (per GL) Actual GL balance / actual hours
WC hours Family reference hours Actual MO operation hours
Price variance = (actual price − std price) × actual qty
Volume variance = (actual qty − std qty) × std price
Efficiency variance = (actual hours − std hours) × std rate
All decomposable by work centre, by GL account, by family.

8. Data Architecture — Where Is What Stored?

Data Model Maintained By When Updated
Production parameters (hours, scrap, chemical) kebony.planning.family Production manager When process changes (rare)
Raw wood & chemical product links kebony.planning.family Production manager Once per family (setup)
Product BOMs (wood, chemical, operations) mrp.bom Auto-generated (Push BoM button) After any family parameter change
WC hourly rates (by GL cost nature) kebony.cost.version.wc.line Controller / CFO Monthly (new cost version)
RM standard prices (wood, chemical) kebony.cost.version.product.line Controller / Purchasing Monthly (new cost version)
BU budgets (top-down) kebony.cost.version.bu.line CFO / Finance Monthly or quarterly
Family cost card (std cost per m³) kebony.cost.version.family.line Auto-computed On any input change (stored)
Product cost card (std cost per m³) product.template (computed) Auto-computed Real-time (non-stored)
Actual production costs mrp.production + FIFO layers System (MO completion) At MO closing
Golden rule: Parameters live on the family. BOMs are generated, not manually maintained. Prices & rates live on the cost version. The cost card is computed. Never edit downstream — always change the source.

9. Maintenance Workflow — Who Does What, When

Production Manager
  • Update family parameters when process changes
  • Set raw_wood_product_id per family (once)
  • Click "Push BoM to Products" after changes
  • Assign planning family to new products
  • Review scrap % quarterly with quality team
Frequency: Rare — only when a production process actually changes (new recipe, equipment upgrade, different chemical mix).
Controller / CFO
  • Create new cost version each month
  • Duplicate previous month → adjust
  • Update WC rates if cost pool changes
  • Update RM prices from latest PO data
  • Review family cost card totals
  • Lock & Approve when satisfied
Frequency: Monthly — part of EOM close process. Lock before running margin reports.
System (Automatic)
  • Family cost card: auto-computed on input change
  • Product cost card: auto-computed from active version
  • BOM wood qty: auto from scrap factor
  • BOM operations: auto from family hours
  • BOM chemical: auto from mix consumption
  • FIFO layers: auto at goods receipt
  • MO actual cost: auto at MO completion

10. Chemical Supply Chain — Separate Internal Flow

The chemical mix has its own internal supply chain. The autoclave never consumes raw chemicals directly — only ready-mix.

Flow
Raw Chemicals (FIFO valued) Water: 17,680 liters Furfuryl Alcohol: 9,821 kg Maleic Anhydride: 250 kg Citric Acid: 147 kg Sodium Carbonate: 145 kg ↓ Mix Production MO Consumes raws → Produces Ready-Mix 1 batch = 28,043 kg ↓ Ready-Mix Tank (stock location) Triggered by minimum tank level ↓ Autoclave Consumption BOM line: mix_consumption_per_m³ (kg)
Costing Implication

Ready-Mix Valuation

FIFO — each Mix MO batch creates a FIFO layer at actual raw chemical cost + absorbed WC (Chemical Mixing work centre).

Standard Chemical Cost

Cost version stores a ready_mix_price_per_kg via the Product Price line. Family cost card computes: mix_consumption × price = chemical cost per m³.

Variance

Actual ready-mix FIFO cost vs standard price = chemical price variance. Actual kg consumed per m³ vs standard = chemical efficiency variance.

11. The Full Picture

MASTER DATA (rarely changes) Planning Family → hours, scrap %, chemical rate, raw_wood_product, ready_mix_product Work Centres → Stacking, Autoclave, Dryer, Chemical Mixing (physical definitions) BUDGET LAYER (monthly, by controller) Cost Version → period + version_type + locked status ├ BU Budget Lines → BU (analytic) + GL account + planned € ├ WC Cost Lines → WC + GL account + €/hour ← THE CLOSED LOOP ├ Product Prices → product + standard price / UoM └ Family Cards → hours + scrap snapshot + computed cost per m³ BOM LAYER (auto-generated from family) Push BoM to Products → wood (scrap-adjusted) + chemical + 3 WC operations One BOM per product, template from family OPERATIONAL LAYER (production) Dryer Load → Autoclave Batches → Process Packs → MO (Phase 6) Actual hours, actual consumption, actual FIFO cost GAP ANALYSIS Budget (cost version) vs Actual (MO + FIFO) → Per product, per family, per WC, per GL account → Price / Volume / Efficiency variance decomposition
Everything is connected. Change a WC rate → family cost card updates → product cost card updates. Change scrap % → BOM wood qty updates → cost card updates. Lock the version → snapshot frozen for gap analysis. FIFO provides the truth. The cost version provides the benchmark.
