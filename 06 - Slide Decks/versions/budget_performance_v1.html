Domain 1 — Sales Budget

Revenue + volume + GM1 targets per country, per month

Budget entry dimensions

Field Description
Company Kebony INC or BNV
Country Destination country
Sales Area US only (optional). BP2026: single catch-all. BP2027+: per area
Period Calendar month
Revenue Target in local currency (USD / EUR)
Volume m3 Universal physical volume
Volume LF / LM Linear feet (US) or meters (BE)
COGS Expected cost of goods sold
GM1 % Gross margin target percentage
US Sales Areas: model supports per-area budgets. For BP 2026, all US volume maps to a single "US National" area. Team can start declining per area for BP 2027.

Actuals — from existing data

No double entry needed

  • Revenue ← posted invoices by country/month
  • Volume m3 ← x_kebony_total_volume_m3 on invoice
  • COGS ← x_studio_cogs on invoice header
  • GM1 = Revenue - COGS (computed)
  • GM1 % = GM1 / Revenue (computed)

Budget Versions & Reforecast

Each company owns its versions — BP, RF1, RF2…

Version model

Field Example
Name BP 2026 — Kebony INC
Fiscal year 2026
Type Budget or Reforecast
Company Kebony INC
State Draft → Approved → Locked

Lifecycle

Draft ──► Approved ──► Locked │ └──► Create RF1 (copy, modify, approve) └──► Create RF2 …

Dashboard shows

  • BP — original annual budget (locked)
  • RF — latest reforecast (if any)
  • Actual — computed from Odoo data
  • LY — prior year actuals (from 2027)
  • Variance % — actual vs BP (or RF)

Data entry workflow

Annual Budget (BP)

  1. Finance creates version for each entity
  2. Sales: revenue + volume + GM1 per country per month
  3. Production: m3 + dryer loads per chain per month
  4. Expenses: amounts per GL + cost center per month
  5. Cash flow: WCR assumptions, capex, financial costs
  6. Stock targets: m3 + days per location
  7. FTE: headcount per BU
  8. KPIs: OTIF target, DSO target
  9. Approve and lock

Excel Import

Template Excel per domain. Import wizard validates and creates lines. Avoids tedious manual entry.

Actuals are never entered — they are computed live from Odoo's accounting, inventory, manufacturing, and sales data.

Implementation Plan

4 weeks development + validation post go-live
Week 1 — Models + Forms

Development

  • All 8 domain models
  • Form & list views
  • Excel import wizard
  • Security groups + menus
  • Extend cost version BU lines

Straightforward Odoo model work — Float fields + foreign keys.

Weeks 2–3 — Controller Input

Finance / Controller

  • GL mapping for EBITDA waterfall
  • GM1 definition confirmation
  • Stock category rules
  • Cash flow opening balances
  • Actual BP numbers (Excel)

Dev does NOT block — models are ready, waiting for data.

Week 4 — Actuals + Dashboard

Development

  • SQL views for all actuals
  • EBITDA GL mapping
  • KPI actuals (OTIF, DSO)
  • Backlog + Top 10 views
  • BP vs Actual PIVOT views
  • Odoo Spreadsheet dashboard
Post Go-Live — Validation

Joint

  • US-only initially
  • Need real data flowing
  • Dashboard review with CEO
  • Belgium onboarded later

Validation requires live transactional data — cannot happen before go-live.

Future — CRM

Arch. projects in pipe + pipeline value. When CRM is implemented.

Total: ~4 weeks development + validation post go-live. Key dependency: controller providing GL mapping and BP numbers (weeks 2-3). Dev work continues in parallel.

Open Questions

Input needed from Finance & Operations before implementation

1. GM1 Definition

Is GM1 = Revenue - COGS only?
Or does it include accruals (marketing, royalties, commissions)?

2. Stock Categories

Three categories: RM, Semi-Finished, FG.
Do we need a 4th for consignment stock (Biewer)?

3. Budget Approval

Who approves budget versions?
Single approver or multi-level workflow?

4. Currency Scope

Belgium budgets in EUR only?
Or also NOK for the Norway entity?

5. EBITDA GL Mapping

Which GL accounts go into D&A add-back?
Need the account list from finance.

6. DSO Scope

US only or also Belgium?
BE has 90% factoring + prepayment — less sensitive but still trackable.

7. Production Budget Owner

Operations team or finance enters production targets?

8. Cash Flow Opening Balance

Manual entry for month 1 of each fiscal year?
Or pull from Odoo's bank reconciliation?
