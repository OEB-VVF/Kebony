Proposed Solution — Three Fields, Three Roles

One new field, one relabel, one unchanged.
Field Label Technical Name Type Contains Action
Sales Responsible x_kebony_sales_responsible_id res.users Kebony employee who owns the deal NEW — create
Back Office Person user_id res.users Admin staff handling the order RELABEL from “Salesperson”
External Sales Agent x_studio_sales_representative res.partner Agent company US only KEEP as-is

Migration

Step 1 — Relabel

Rename user_id from “Salesperson” to “Back Office Person”. No data change — just a clearer label.

Step 2 — New field

Create x_kebony_sales_responsible_id (Many2one → res.users) on sale.order and account.move. Required on SO confirmation.

Step 3 — Propagation
Sales Order → Invoice → Credit Note → Reports

Sales Responsible flows automatically through the entire document chain.

The external sales agent field stays untouched — it is correctly used for US agent commission tracking.
